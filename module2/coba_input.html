Uji coba 7





Berdasarkan percobaan yang dilakukan, atribut min dan max pada input HTML berfungsi untuk membatasi nilai yang dapat dimasukkan oleh pengguna. Saat nilai min pada input range diubah menjadi -50, slider dapat digeser ke angka negatif tanpa terjadi error. Hal ini menunjukkan bahwa browser secara otomatis menyesuaikan batas nilai sesuai atribut yang diberikan. Dengan demikian, atribut min dan max sangat penting untuk mengontrol rentang input agar sesuai dengan kebutuhan sistem.

Uji coba 8







Berdasarkan percobaan yang dilakukan di beberapa browser, setiap tipe input HTML memiliki fungsi validasi bawaan. Input type="url" hanya menerima format alamat website yang valid dan akan menampilkan peringatan jika format salah. Input type="search" berfungsi seperti text biasa namun memiliki fitur tambahan seperti tombol clear di beberapa browser. Sedangkan input type="tel" dengan atribut pattern membatasi format nomor telepon sesuai pola yang ditentukan (contoh: 123-45-678). Jika format tidak sesuai, browser akan menolak pengiriman form dan menampilkan pesan error. Hal ini menunjukkan bahwa HTML5 sudah menyediakan validasi dasar tanpa perlu JavaScript.

Uji coba 9





Berdasarkan percobaan yang dilakukan di beberapa browser, input type="time" menampilkan pemilih waktu (jam dan menit) secara otomatis sehingga memudahkan pengguna dalam memilih waktu tanpa harus mengetik manual. Sedangkan input type="week" menampilkan pilihan minggu dalam format tahun dan minggu (YYYY-Wxx). Namun, tidak semua browser mendukung tampilan yang sama. Pada browser modern seperti Chrome dan Edge, fitur ini berjalan dengan baik, tetapi pada beberapa browser lama tampilannya bisa berbeda atau berubah menjadi input teks biasa. Hal ini menunjukkan bahwa dukungan terhadap tipe input HTML5 dapat berbeda tergantung browser yang digunakan.
